
materials installation instructions:
------------------------------------

1 - Third-party software setup :
--------------------------------

materials depends on the following libraries and/or programs :
 - datatools (>=5.0.0)


2- Plain CMake installation :
-----------------------------

Change to the materials source base directory :
{{{
bash$ cd <materials_source_base_dir>
}}}

Create a 'build' directory :
{{{
bash$ mkdir build
bash$ cd build
}}}

Configure the installation to the '${HOME}/software' directory.
Note: you must choose a directory where you have write access.

{{{
bash$ cmake \
   -DCMAKE_INSTALL_PREFIX:PATH=${HOME}/software \
   -DCMAKE_BUILD_TYPE:STRING=Release \
   -Ddatatools_DIR=<path_to_datatools_cmake_config_dir> \
   ..
}}}

This prints :
{{{
-- The C compiler identification is GNU
-- The CXX compiler identification is GNU

...

-- Configuring done
-- Generating done
-- Build files have been written to: <materials base dir>/build
}}}

Build :
{{{
bash$ make
}}}

Generate the documentation :
{{{
bash$ make doc
}}}

Perform tests :
{{{
bash$ make test
}}}

Clean test material :
{{{
bash$ make clean_test
}}}

Installation (here in the '${HOME}/software' directory) :
{{{
bash$ make install
}}}

Remove the temporary build directory :
{{{
bash$ cd build
bash$ rm -fr build
}}}


3 - Production CMake installation using the 'pkgtool' wrapper :
---------------------------------------------------------------

Change to the materials source base directory :
{{{
bash$ cd <materials_source_base_dir>
}}}

Getting help :
{{{
bash$ ./pkgtools.d/pkgtool --help
}}}

Run the following configure/build/install sequence :
{{{
bash$ ./pkgtools.d/pkgtool check
bash$ ./pkgtools.d/pkgtool configure --install-prefix ${HOME}/software
bash$ ./pkgtools.d/pkgtool build --number-of-tasks auto
bash$ ./pkgtools.d/pkgtool doc
bash$ ./pkgtools.d/pkgtool test
bash$ ./pkgtools.d/pkgtool install
}}}

De-installation :
{{{
bash$ ./pkgtools.d/pkgtool uninstall
}}}

Full reset of the package :
{{{
bash$ ./pkgtools.d/pkgtool reset
}}}


4 - Setup :
-----------

The following instructions applies to the bash shell.

Load the definition of the materials setup function :
{{{
bash$ source ${HOME}/software/etc/materials_setup.sh
NOTICE: do_datatools_setup: Please setup 'materials' with 'do_materials_setup'.
}}}

Invoke the setup function :
{{{
bash$ do_materials_setup
NOTICE: do_materials_setup: Setup materials...
NOTICE: do_materials_setup: Setup materials done.
}}}

Using the materials-config utility script :
{{{
bash$ which materials-config
/home/<login>/software/bin/materials-config
bash$ materials-config --version
4.0.0
bash$ materials-config --cflags
...
bash$ materials-config --ldflags
...
}}}
